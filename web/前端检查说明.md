## 三、已做修改（必读）

### 1. 端口冲突

- **问题**：Next.js 默认 `next dev` 使用 3000，与后端 API（3000）冲突。
- **修改**：在 `web/package.json` 中已将 `next dev` 改为 **`next dev -p 3001`**，`next start` 改为 **`next start -p 3001`**。
- **结果**：根目录 `npm run dev` 时，后端 3000，前端 3001，不再冲突。

### 2. API 代理

- **问题**：前端需请求后端 `/api/*`，若直接写 `http://localhost:3000/api` 需处理跨域且写死地址。
- **修改**：在 `web/next.config.mjs` 中增加了 **rewrites**，将前端的 `/api/*` 代理到 `http://localhost:3000/api/*`。
- **结果**：前端代码里请求 `/api/xxx` 即可（同源），开发时自动转到后端。

### 3. API 封装

- **新增**：`web/lib/api.ts`，提供 `api(path, options)` 与 `setToken`/`getToken`，请求自动带 `Authorization: Bearer <token>`，401 时清 token 并跳转。
- **用法**：在需要调后端的页面中 `import { api, setToken } from '@/lib/api'`，用 `api('/sys/org')`、`api('/person/archive', { method: 'POST', body: { ... } })` 等调用，与后端 `server/docs/API.md` 对应。

## 四、当前数据状态与后续对接

- **现状**：PC 端如 `app/pc/dashboard/page.tsx`、`app/pc/personnel/archive/page.tsx` 等使用的是**前端静态 mock 数据**（如 `stats`、`personnelData`），尚未请求后端。
- **后端接口**：已就绪，见 `server/docs/API.md`（如 GET `/api/sys/org`、GET `/api/person/archive`、POST `/api/auth/login` 等）。
- **建议**：
  1. 在 PC 布局或登录流程中接入 **POST /api/auth/login**，登录成功后 `setToken(data.token)`，并跳转到 `/pc/dashboard`。
  2. 各业务页逐步把 mock 数据改为 `api('/person/archive')`、`api('/data/board')` 等，与后端联调。
  3. H5 端同理，使用工号+姓名 **POST /api/auth/worker-login**，用独立 token（可再封装 `lib/workerApi.ts` 存 `labor_worker_token`）。

## 六、安装与运行

```bash
# 在项目根目录
npm run install:all

# 同时启动后端 + 前端
npm run dev
```

- 后端 API：http://localhost:3000  
- 前端页面：http://localhost:3001（首页、/pc/dashboard、/h5 等）

若你本地使用 **pnpm**，可在 `web` 目录执行 `pnpm install` 与 `pnpm dev`，效果一致（端口已改为 3001）。

## 七、其他注意

- **TypeScript**：`next.config.mjs` 中 `ignoreBuildErrors: true`，构建会忽略 TS 错误；建议逐步修掉错误后关闭该选项。
- **ESLint**：已配置 `"lint": "eslint ."`，可定期运行 `npm run lint`（或在 web 下 `pnpm lint`）。
- **图标/静态资源**：layout 中引用了 `/icon-light-32x32.png`、`/apple-icon.png` 等，若缺失可在 `public/` 下补或暂时移除 metadata 中的引用。
