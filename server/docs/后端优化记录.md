# 后端优化记录

> 本节记录近期对后端的集中优化，便于后续维护与扩展。

---

## 1. 数据库

- **索引**（`db/schema.js`）：为常用筛选与关联增加索引，减轻列表/报表查询压力。
  - `person`：`org_id`、`status`、`on_site`
  - `attendance`：`work_date`、`person_id`
  - `contract_instance`：`status`、`person_id`
  - `settlement`：`(person_id, status)`
  - `notification`：`person_id`

---

## 2. 统一响应与错误

- **错误格式**：所有 4xx/5xx 统一为 `{ code, message [, details ] }`，code 如 `UNAUTHORIZED`、`BAD_REQUEST`、`NOT_FOUND`。
- **使用位置**：`auth.js` 登录、`auth` 中间件 401、`requireWorker` 401、`contract.js` 合同发起 400、`settlement.js` 生成 400 等已改用 `lib/response.js` 的 `err(res, status, message)`。
- **全局错误处理**：`middleware/errorHandler.js` 捕获未处理异常，返回 500 与统一 body。

---

## 3. 常量与校验

- **常量**（`lib/constants.js`）：人员状态列表、合同/结算状态、用户角色、分页默认值与上限，避免魔法字符串分散在路由中。
- **校验**（`lib/validate.js`）：`requireNonEmptyString`、`requirePositiveIntArray` 用于必填与数组参数；`parsePagination(query)` 统一解析 `page`/`pageSize` 并限制上限。
- **使用示例**：人员档案列表使用 `parsePagination` 并返回 `page`/`pageSize`；合同发起对 `title`、`person_ids` 做校验。

---

## 4. 异步路由

- **asyncHandler**（`middleware/asyncHandler.js`）：包装异步路由函数，将 Promise 拒绝交给全局 errorHandler，避免未捕获异常导致进程退出。新写 async 路由时可使用：`router.get('/path', asyncHandler(async (req, res) => { ... }))`。
