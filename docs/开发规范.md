# 数字劳务 · 开发规范

便于团队协作与后期维护的约定（目录、接口、提交等）。

---

## 一、目录与职责

| 目录/文件 | 职责 |
|-----------|------|
| **web/** | 前端工程（Vite + React）。管理端 /admin、工人端 /h5 同仓，构建产物为静态资源。 |
| **server/** | 后端工程（Express）。仅提供 REST API（/api/*），不托管前端静态资源。 |
| **docs/** | 项目级文档：部署说明、开发规范、可选架构说明副本。 |
| **功能点/** | 需求与报价相关文档，产品/对接用。 |
| **.env.example** | 环境变量示例，复制为 .env 后本地修改（.env 不提交）。 |

- 前后端通过 **HTTP API** 通信，接口约定见 `server/docs/API.md`。
- 新增接口时请同步更新 `server/docs/API.md`。

---

## 二、后端约定

- **技术栈**：Node.js 18+、Express、SQLite（better-sqlite3）、JWT。
- **错误响应**：4xx/5xx 统一 body `{ code, message [, details ] }`，见 `server/lib/response.js`。
- **分页**：列表类接口使用 `page`、`pageSize`，上限 100，见 `server/lib/validate.js` 的 `parsePagination`。
- **常量**：业务枚举（状态、角色等）放在 `server/lib/constants.js`。
- **测试**：在 `server` 目录执行 `npm test`，新增接口建议补充用例。

---

## 三、前端约定

- **技术栈**：Vite、React 18、React Router 6。
- **请求**：管理端使用封装后的 api（带 labor_token）；工人端使用 workerApi（带 labor_worker_token）。基地址开发时通过 Vite 代理指向后端。
- **路由**：/admin/* 管理端，/h5/* 工人端；未登录 401 时跳转对应登录页。

---

## 四、环境与分支（建议）

- 开发环境：本地 `npm run dev`，前端代理到本地后端。
- 生产环境：后端通过 PORT、JWT_SECRET 等环境变量配置；前端构建时可按环境注入 API 基地址。
- 分支建议：main 对应生产可部署版本；开发在 feature/* 或 dev，按需合并。

---

## 五、提交与版本

- 提交信息建议简洁说明改动范围（如「后端：合同发起增加校验」「前端：登录页样式」）。
- 大功能或破坏性变更请在 README 或 docs 中注明。

---

**文档版本**：与当前项目一致；规范若有调整请同步更新。
