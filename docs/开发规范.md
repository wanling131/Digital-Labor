# Digital Labor · 开发规范

> 本文档说明团队协作与代码编写的约定，确保项目可维护性。

---

## 一、代码规范

### 1.1 后端规范

- **错误响应**: 4xx/5xx 统一返回 `{ code, message [, details] }`
- **分页**: 使用 `page`、`pageSize`，上限 100
- **常量**: 业务枚举放在 `server/lib/constants.js`
- **测试**: 新增接口建议补充测试用例

### 1.2 前端规范

- **组件**: 使用函数组件 + Hooks
- **样式**: Tailwind CSS，避免内联样式
- **类型**: TypeScript 严格模式
- **请求**: 管理端用 `api()`，工人端用 `apiWorker()`

### 1.3 Git提交

- 提交信息格式: `[范围] 描述`，如 `后端: 合同发起增加校验`
- 范围可选: 后端、前端、文档、配置
- 大功能请在提交信息中注明影响范围

## 二、接口规范

### 2.1 RESTful设计

| 操作 | 方法 | 路径示例 |
|------|------|----------|
| 查询列表 | GET | `/api/person/archive` |
| 查询详情 | GET | `/api/person/archive/:id` |
| 创建 | POST | `/api/person/archive` |
| 更新 | PUT | `/api/person/archive/:id` |
| 删除 | DELETE | `/api/person/archive/:id` |

### 2.2 响应格式

**成功**:
```json
{ "data": {} }
// 或
{ "list": [], "total": 100 }
```

**失败**:
```json
{ "code": "ERROR_CODE", "message": "错误描述" }
```

## 三、数据库规范

### 3.1 表命名

- 小写 + 下划线，如 `contract_instance`
- 复数形式，如 `users`、`contracts`

### 3.2 字段命名

- 小写 + 下划线
- 时间字段: `created_at`、`updated_at`
- 状态字段: 明确枚举值，如 `status: 'pending' | 'approved' | 'rejected'`

### 3.3 索引

- 外键字段必须加索引
- 常用查询字段加索引
- 索引命名: `idx_表名_字段名`

---

## 四、安全规范

### 4.1 认证

- JWT Token 设置合理过期时间
- 敏感操作需二次验证
- Token 泄露处理: 加入黑名单

### 4.2 数据

- 密码必须加密存储
- 敏感信息脱敏展示
- SQL 注入防护: 使用参数化查询

### 4.3 文件

- 上传文件类型白名单
- 文件大小限制
- 存储路径不暴露

---

## 五、性能规范

### 5.1 后端

- 列表接口必须分页
- 大数据查询加缓存
- N+1 查询优化

### 5.2 前端

- 图片懒加载
- 组件按需加载
- 避免重复请求
