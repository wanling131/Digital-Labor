# 前后端数据对接说明

本文档说明前端虚拟数据与后台数据库的对接方式：数据存入数据库，通过 API 获取并展示。

## 一、数据库与种子数据

- **库文件**：`server/data/labor.db`（SQLite）
- **表结构**：见 `server/db/schema.js`
- **种子数据**：首次启动或执行 `node server/scripts/seed.js` 时，会写入组织、人员、合同、考勤、结算、通知、操作日志等示例数据
- **组织表**：新增 `manager` 字段（负责人），组织树接口返回每节点 `memberCount`（含子节点人数）

## 二、PC 管理端

| 页面       | 原数据来源 | 现数据来源 | 接口 |
|------------|------------|------------|------|
| 组织管理   | 前端 mock 树 | 数据库     | `GET /api/sys/org`（树含 memberCount、manager） |
| 角色权限   | 前端 mock 角色列表 | 数据库     | `GET /api/sys/role`（含 userCount） |
| 操作日志   | 前端 mock 日志列表 | 数据库     | `GET /api/sys/log?page=1&pageSize=20`（分页） |

- PC 端请求使用管理端 Token（`labor_token`），由 `web/lib/api.ts` 的 `api()` 统一携带。

## 三、H5 工人端

| 页面/模块   | 原数据来源 | 现数据来源 | 接口 |
|-------------|------------|------------|------|
| 首页        | 前端 mock 通知、姓名/组织 | 数据库     | `GET /api/notify/list`、`GET /api/worker/me` |
| 我的合同    | 前端 mock 合同列表 | 数据库     | `GET /api/contract/my-pending`、`GET /api/contract/my-signed`，签署 `POST /api/contract/sign/:id` |
| 工资查询    | 前端 mock 工资记录 + 待确认 | 数据库     | `GET /api/settlement/my-pending`、`GET /api/settlement/my`，确认/驳回 `POST /api/settlement/confirm/:id` |
| 考勤打卡    | 前端 mock 考勤记录 | 数据库     | `GET /api/attendance/my?year=&month=` |
| 消息通知    | 前端 mock 通知列表 | 数据库     | `GET /api/notify/list`，已读 `PUT /api/notify/:id/read` |
| 个人中心    | 前端写死姓名/工号等 | 数据库     | `GET /api/worker/me` |

- H5 端请求使用工人端 Token（`labor_worker_token`），由 `web/lib/api.ts` 的 **`apiWorker()`** 统一携带；401 时跳转 `/h5/login`。

## 四、前端 API 封装

- **`api(path, options)`**：管理端，带 `labor_token`，401 跳 `/login`。
- **`apiWorker(path, options)`**：工人端，带 `labor_worker_token`，401 跳 `/h5/login`。
- 支持 `query` 参数：`api('/api/sys/log', { query: { page: 1, pageSize: 20 } })`。
- **`downloadContractPdf(contractId, useWorkerToken?)`**：下载合同 PDF，H5 传 `true` 使用工人 Token。

## 五、使用与验证

1. **启动后端**：`cd server && npm run dev`（默认 3000 端口）
2. **启动前端**：`cd web && npm run dev`（默认 3001）
3. **管理端**：访问 `/login` 登录（admin / 123456），组织、角色、操作日志等均从接口拉取
4. **工人端**：访问 `/h5/login` 用工号+姓名登录（须与 `person` 表中 `work_no`、`name` 一致），首页、合同、工资、考勤、消息、个人中心均从接口拉取
5. **重新初始化数据**：`cd server && node scripts/seed.js`（会清空业务数据后重新种子）

## 六、数据流小结

- 种子数据写入 **org、user、person、contract_template、contract_instance、attendance、settlement、notification、op_log** 等表。
- PC 管理端：组织树、角色（含人数）、操作日志均来自上述表，通过 `/api/sys/*` 等接口返回。
- H5 工人端：合同、结算、考勤、通知、当前工人信息均来自上述表，通过 `/api/contract/*`、`/api/settlement/*`、`/api/attendance/my`、`/api/notify/*`、`/api/worker/me` 返回，且均需工人 Token（工号+姓名登录后下发）。
