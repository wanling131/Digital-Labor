## 一、项目概述

### 1.1 系统架构

Digital Labor 是一套面向建筑劳务行业的数字化管理系统，采用前后端分离架构：

```
┌─────────────────────────────────────────────────────────────┐
│                    Digital Labor 系统                      │
├─────────────────────────────────────────────────────────────┤
│  PC管理端 (Next.js)    │  H5工人端 (Next.js)      │
│  - 人员档案管理        │  - 个人中心               │
│  - 合同管理            │  - 我的合同               │
│  - 考勤管理            │  - 我的考勤               │
│  - 结算管理            │  - 我的薪资               │
│  - 现场管理            │  - 消息通知               │
│  - 系统管理            │  - 激活流程               │
├─────────────────────────────────────────────────────────────┤
│              后端服务 (Node.js + Express)                │
│  - 认证授权  - 业务API  - 数据处理  - 文件管理       │
├─────────────────────────────────────────────────────────────┤
│              数据库 (SQLite)  +  文件存储               │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Next.js 16 + React 19 + TypeScript | PC管理端和H5工人端 |
| 后端 | Node.js 18+ + Express | REST API服务 |
| 数据库 | SQLite (better-sqlite3) | 本地文件数据库 |
| UI组件 | Radix UI + Tailwind CSS | 组件库和样式 |
| 图标 | Lucide React | 图标库 |

### 1.3 项目结构

```
digital-labor/
├── docs/                    # 项目文档
├── scripts/                 # 脚本工具
├── server/                  # 后端服务
│   ├── db/                  # 数据库
│   ├── middleware/          # 中间件
│   ├── routes/              # 路由
│   └── test/                # 测试
├── web/                     # 前端应用
│   ├── app/                 # 页面路由
│   ├── components/          # 组件
│   └── lib/                 # 工具库
├── 功能点/                   # 需求文档
├── package.json             # 根目录配置
└── README.md                # 项目总览
```

## 三、接口对接

### 3.1 认证与登录

| 端 | 入口 | 演示账号 |
|----|------|----------|
| PC管理端 | `/login` | admin / 123456 |
| H5工人端 | `/h5/login` | 工号+姓名（需存在于person表） |

**Token机制**:
- PC端: `labor_token` → 401跳转 `/login`
- H5端: `labor_worker_token` → 401跳转 `/h5/login`

### 3.2 核心接口列表

#### 认证相关
- `POST /api/auth/login` - 管理端登录
- `POST /api/auth/worker-login` - 工人端登录

#### 人员档案
- `GET/POST/PUT/DELETE /api/person/archive` - 人员档案CRUD
- `GET /api/person/auth` - 认证列表
- `PUT /api/person/:id/auth-review` - 审核
- `GET /api/person/status` - 状态统计
- `POST /api/person/status/batch` - 批量变更

#### 合同管理
- `GET/POST /api/contract/template` - 合同模板
- `POST /api/contract/launch` - 合同发起
- `GET /api/contract/status` - 签约状态
- `GET /api/contract/archive` - 合同归档
- `POST /api/contract/sign/:id` - 签署合同

#### 考勤管理
- `POST /api/attendance/import` - Excel导入
- `GET /api/attendance/report` - 工时报表
- `GET /api/attendance/my` - 我的考勤

#### 结算管理
- `POST /api/settlement/generate` - 生成结算单
- `GET /api/settlement/confirm` - 待确认列表
- `POST /api/settlement/confirm/:id` - 确认/驳回
- `GET /api/settlement/my` - 我的工资

#### 系统管理
- `GET/POST/PUT /api/sys/user` - 用户管理
- `GET/POST/PUT/DELETE /api/sys/org` - 组织管理
- `GET /api/sys/log` - 操作日志
- `GET /api/sys/my-menu` - 我的菜单

#### H5工人端
- `GET/PUT /api/worker/me` - 个人信息
- `GET /api/contract/my-pending` - 待签署合同
- `GET /api/contract/my-signed` - 已签署合同
- `GET /api/notify/list` - 通知列表
- `PUT /api/notify/:id/read` - 标记已读

### 3.3 数据对接说明

**数据库**: `server/data/labor.db` (SQLite)

**种子数据**: 首次启动自动导入示例数据

**主要数据表**:
- `org` - 组织架构
- `user` - 管理端用户
- `person` - 人员档案
- `contract_template` - 合同模板
- `contract_instance` - 合同实例
- `attendance` - 考勤记录
- `settlement` - 结算单
- `notification` - 通知
- `op_log` - 操作日志

## 四、开发规范

### 4.1 目录职责

| 目录 | 职责 |
|------|------|
| `web/` | 前端工程（Next.js） |
| `server/` | 后端工程（Express） |
| `docs/` | 项目文档 |
| `功能点/` | 需求与报价文档 |

### 4.2 后端约定

- **错误响应**: 4xx/5xx统一返回 `{ code, message [, details] }`
- **分页**: 使用 `page`、`pageSize`，上限100
- **常量**: 业务枚举放在 `server/lib/constants.js`
- **测试**: `npm test` 运行接口测试

### 4.3 前端约定

- **请求**: 管理端用 `api()`，工人端用 `apiWorker()`
- **路由**: `/pc/*` 管理端，`/h5/*` 工人端
- **401处理**: 自动跳转对应登录页

### 4.4 Git提交规范

- 提交信息简洁说明改动范围
- 大功能或破坏性变更请在README中注明

## 五、部署配置

### 5.1 环境变量

创建 `.env` 文件（已加入.gitignore）：

```bash
# 后端服务端口（默认3000）
PORT=3000

# JWT密钥（生产环境务必修改）
JWT_SECRET=labor-secret-change-in-prod

# 运行环境
NODE_ENV=development

# 第三方服务（可选）
ALIYUN_ACCESS_KEY_ID=your_key
ALIYUN_ACCESS_KEY_SECRET=your_secret
ESIGN_APP_ID=your_app_id
ESIGN_APP_SECRET=your_secret
```

### 5.2 开发环境启动

```bash
# 安装所有依赖
npm run install:all

# 同时启动后端+前端
npm run dev

# 单独启动
npm run dev:server  # 后端: http://localhost:3000
npm run dev:web     # 前端: http://localhost:3001
```

### 5.3 生产部署

**后端**:
```bash
cd server
npm install --production
npm run start
# 或使用pm2
pm2 start server.js -n labor-api
```

**前端**:
```bash
npm run build
# 将web/dist部署到Nginx或静态托管
```

**Nginx配置示例**:
```nginx
server {
  listen 80;
  server_name your-domain.com;
  root /path/to/web/dist;
  index index.html;
  
  location / {
    try_files $uri $uri/ /index.html;
  }
  
  location /api/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }
}
```
