# 数字劳务 · 测试说明

本文说明如何**分别测试后端、前端**，以及如何做**前后端连通性测试**和**统一测试**。

---

## 一、后端单独测试

在项目根目录执行：

```bash
npm run test:server
```

或进入后端目录执行：

```bash
cd server && npm test
```

- **内容**：Node 内置 test + supertest，覆盖健康检查、登录、鉴权、组织、人员、看板、工人登录、my-menu、push-notify 等接口。
- **预期**：全部用例通过（当前约 16 条）。
- **说明**：不启动端口，直接对 Express app 发请求，无需先启动服务。

---

## 二、前端单独测试

在项目根目录执行：

```bash
npm run test:web
```

或进入前端目录执行：

```bash
cd web && npm run build
```

- **内容**：Next.js 生产构建（`next build`），校验 TypeScript、依赖与页面能否正常构建。
- **预期**：构建成功，无报错。
- **说明**：不启动 dev server，仅做构建校验；若使用 pnpm，可在 web 目录执行 `pnpm build`。

---

## 三、前后端连通性测试

校验「后端服务已启动时，接口是否可访问」以及「前端服务已启动时，首页是否可访问」。

### 1. 先启动后端（必须）

在一个终端执行：

```bash
npm run dev:server
```

保持运行，确认控制台出现 `API http://localhost:3000`。

### 2. 可选：启动前端

在另一个终端执行：

```bash
npm run dev:web
```

保持运行，确认前端在 http://localhost:3001。

### 3. 执行连通性测试

在**第三个终端**（或先停掉 dev 再在同一个终端跑）执行：

```bash
npm run test:connectivity
```

或直接运行脚本：

```bash
node scripts/connectivity-test.mjs
```

- **检查项**：
  - 后端健康检查：GET http://localhost:3000/api/health
  - 后端元信息：GET http://localhost:3000/api
  - 管理端登录：POST http://localhost:3000/api/auth/login（admin / admin123）
  - 前端首页：GET http://localhost:3001（若前端未启动则此项不通过，不影响后端结论）
- **预期**：后端三项通过即退出码 0；前端未启动时仅提示「前端连通性: 未通过」。
- **环境变量**：可通过 `API_BASE`、`WEB_BASE` 修改地址，例如：
  - `API_BASE=http://127.0.0.1:3000 WEB_BASE=http://127.0.0.1:3001 node scripts/connectivity-test.mjs`

---

## 四、统一测试（不启动服务）

在项目根目录执行：

```bash
npm run test:all
```

- **内容**：先执行 `npm run test:server`（后端单元测试），再执行 `npm run test:web`（前端构建）。
- **预期**：两步都成功则退出码 0；任一步失败则终止并返回非 0。
- **说明**：不启动任何服务，适合 CI 或本地快速回归。不包含「连通性测试」；连通性需在服务已启动时单独执行 `npm run test:connectivity`。

---

## 五、推荐流程

| 场景 | 操作 |
|------|------|
| 日常开发前自检 | `npm run test:server`，再按需 `npm run test:web` |
| 提交前 / CI | `npm run test:all` |
| 联调前后端 | 先 `npm run dev:server`，再 `npm run dev:web`，最后在另一终端 `npm run test:connectivity` |
| 只验证后端接口 | `npm run test:server` |
| 只验证前端能构建 | `npm run test:web` |

---

**文档版本**：与当前根目录 package.json 脚本一致。
