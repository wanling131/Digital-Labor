## 一、本次优化项（2024-03-24）

| 项 | 说明 |
|----|------|
| 安全加固 | 密码 bcrypt 哈希存储、密码强度验证、登录失败锁定机制（5次/5分钟）、敏感字段加密 |
| 性能优化 | 数据库索引优化（35+ SQLite, 39+ PostgreSQL）、SQL IN 子句参数绑定修复 |
| 权限完善 | 所有管理端列表接口添加 `actor_org_id` 组织树过滤 |
| 审计增强 | `op_log` 表添加 `data_before`/`data_after` 字段，支持数据变更前后快照 |
| 电子签预留 | 抽象 `SignProvider` 接口，预留 e签宝对接能力 |
| 代码质量 | 前端 TypeScript 类型定义（28 interfaces）、输入验证工具、开发工作流规范 |

### 1.1 安全加固详情

| 功能 | 实现 |
|------|------|
| 密码哈希 | 使用 bcrypt 存储，向后兼容明文密码迁移 |
| 密码强度 | 要求 8+ 字符、大小写字母、数字 |
| 登录锁定 | 5 次失败后锁定 5 分钟 |
| 敏感字段 | id_card、mobile、bank_card 加密存储 |
| SQL 注入防护 | IN 子句使用 `bindparam(expanding=True)` |

## 四、已具备的完整链路（可直接验收）

- **PC**：登录 → 组织/用户/人员档案/状态管理 → 考勤导入/工时报表 → 结算生成与确认/薪资报表 → 合同模板/发起/签约状态/归档 → 离场登记/在岗看板 → 数据看板 → 权限分配/操作日志。
- **H5**：工号或手机号登录 → 工作台（考勤、合同、工资、通知、工种/地址/本月统计）→ 考勤打卡与记录（含月份切换）→ 合同待签/已签与签署 → 工资待确认与历史、工资条下载 → 消息中心 → 个人中心（档案、证书、银行卡、编辑、退出）。
