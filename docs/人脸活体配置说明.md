# 人脸/活体配置说明

> 本系统人脸核验已接入 **阿里云视觉智能开放平台 - 人脸 1:N**（创建人脸库、添加人脸样本、添加人脸数据、人脸搜索）。配置 AccessKey 后自动启用真实接口，未配置时使用本地 mock（便于开发与演示）。文档索引见 [docs/README.md](./README.md)。
>
> 参考：[人脸搜索 1:N 接入流程](https://help.aliyun.com/zh/viapi/use-cases/face-search-1-n)、[创建人脸数据库](https://help.aliyun.com/zh/viapi/use-cases/create-a-face-database-1)、[添加人脸样本](https://help.aliyun.com/zh/viapi/use-cases/add-the-face-samples)、[添加人脸数据](https://help.aliyun.com/zh/viapi/use-cases/add-the-face-data-1)、[人脸搜索](https://help.aliyun.com/zh/viapi/use-cases/the-face-of-a-search-1)。

## 一、配置方式概览

| 方式 | 说明 |
|------|------|
| **已配置密钥** | 在 `backend_dtcloud/.env` 配置 `ALIBABA_CLOUD_ACCESS_KEY_ID`、`ALIBABA_CLOUD_ACCESS_KEY_SECRET`（或 `ALIYUN_ACCESS_KEY_ID`/`ALIYUN_ACCESS_KEY_SECRET`）后，人脸请求将走阿里云人脸 1:N |
| **未配置** | 使用内置 mock，始终返回“通过”，便于联调与演示 |
| **可选** | `ALIYUN_FACE_DB_NAME` 人脸库名（默认 `digital_labor_face`）；`ALIYUN_FACE_CONFIDENCE_THRESHOLD` 核验置信度阈值（默认 60.48） |

## 二、阿里云侧准备

### 2.1 开通产品

1. 登录 [阿里云控制台](https://www.aliyun.com/)
2. 开通 **人脸人体**（视觉智能开放平台 - Face Body）
   - 文档：<https://help.aliyun.com/zh/viapi/use-cases/face-search-1-n>
   - 开通：<https://vision.aliyun.com/facebody>
3. 人脸 1:N 中仅 **SearchFace（人脸搜索）** 收费，CreateFaceDb / AddFaceEntity / AddFace 免费

### 2.2 创建 RAM 子账号并授权

1. 进入 **访问控制 RAM** → **用户** → **创建用户**
2. 勾选 **OpenAPI 调用**，保存并记录 **AccessKey ID**、**AccessKey Secret**
3. 为该用户授权：**AliyunVIAPIFullAccess**（视觉智能开放平台完整权限），见 [RAM 授权](https://help.aliyun.com/zh/viapi/developer-reference/use-ram-policy-access-control)

### 2.3 获取密钥

- **AccessKey ID**：形如 `LTAI5t***`（约 24 位字母数字），在 RAM 用户创建时勾选「OpenAPI 调用」后生成
- **AccessKey Secret**：与 AccessKey ID 成对出现，创建时仅展示一次，请妥善保存

### 2.4 和「用户名/密码」的区别

本系统**只认阿里云 RAM 的 AccessKey 对**，不认控制台登录用的「用户名/密码」，也不认其他系统给的「userName + password」或「API Key」。

| 您可能拿到的 | 是否用于本系统 |
|--------------|----------------|
| 阿里云控制台登录账号、密码 | 否，仅用于登录控制台 |
| 其他系统提供的 `userName: digital-labor-app`、`password: AP:xxx` | 否，格式与阿里云 OpenAPI 不兼容 |
| RAM 用户 → 创建 AccessKey 得到的 **AccessKey ID** + **AccessKey Secret** | 是，填到下方 `.env` 中 |

若您只有「userName / password」形式的凭证，需要到 **阿里云控制台 → 访问控制 RAM → 用户** 中为相应 RAM 用户**创建 AccessKey**，才会得到 `ALIYUN_ACCESS_KEY_ID` 和 `ALIYUN_ACCESS_KEY_SECRET`。实人认证等 OpenAPI 调用必须使用这对密钥做签名，无法用用户名+密码替代。

## 三、本系统环境变量配置

### 3.1 后端（backend_dtcloud）

在 **backend_dtcloud** 目录下创建或编辑 `.env` 文件（不要提交到 Git）：

```bash
# 阿里云人脸 1:N（视觉智能开放平台，与官方文档一致）
ALIBABA_CLOUD_ACCESS_KEY_ID=你的AccessKeyId
ALIBABA_CLOUD_ACCESS_KEY_SECRET=你的AccessKeySecret

# 兼容旧配置（二选一即可）
# ALIYUN_ACCESS_KEY_ID=你的AccessKeyId
# ALIYUN_ACCESS_KEY_SECRET=你的AccessKeySecret

# 可选：人脸库名称，默认 digital_labor_face（小写字母/数字/下划线，1~64 位）
# ALIYUN_FACE_DB_NAME=digital_labor_face

# 可选：核验置信度阈值 0~100，默认 60.48（文档建议 60.48/67.87/72.62 对应不同误识率）
# ALIYUN_FACE_CONFIDENCE_THRESHOLD=60.48
```

### 3.2 配置生效方式

- 使用 `python -m digital_labor.run` 启动时，会自动加载 `backend_dtcloud` 目录下的 `.env`
- 若使用 pm2、Docker 等，请在对应环境中配置上述变量

## 四、验证是否生效

### 4.1 看启动日志

Python 后端不会在启动时打印人脸服务类型，请以 **4.2 调接口查询** 为准。

### 4.2 调接口查询

请求（需带管理端或工人端 Token）：

```http
GET /api/sys/feature-status
```

响应示例：

```json
{
  "faceVerify": "aliyun"
}
```

或（未配置密钥时）：

```json
{
  "faceVerify": "mock"
}
```

前端可根据 `faceVerify === 'aliyun'` 显示“已接入真人脸”等标识。

### 4.3 实际核验流程

- **H5 激活 / 个人中心 / 合同签署前 / 结算确认**：进入人脸验证 → 允许摄像头 → 拍照后由前端将 base64 提交到 `POST /api/person/face-verify`（body：`mode: "living"`, `image`: base64, `person_id`: 人员 ID）。
- **已配置密钥时**：
  - **首次**（该人员尚未 `face_verified`）：在阿里云创建人脸库（若不存在）→ 为该人员创建人脸样本（entity_id=person_id）→ 添加本次拍照为人脸数据 → 标记人员已认证。
  - **再次核验**（该人员已 `face_verified`）：用本次拍照在阿里云人脸库中 1:N 搜索 → 若最高分命中为该人员且置信度 ≥ 阈值则通过，并刷新认证状态。
- **未配置密钥时**：走 mock，直接返回通过。

## 五、接口说明（当前实现）

| 模式 | 说明 | 请求体要点 |
|------|------|------------|
| **living** | 活体/人脸 1:N（录入或核验） | `mode: "living"`, `image`: 拍照 base64, `person_id` 必填 |
| **compare** | 1:1 人脸比对 | 当前仍为 mock；`mode: "compare"`, `image`, `target_image`, `person_id` |
| **full** | 实人认证（姓名+身份证+MetaInfo） | 当前仍为 mock；`mode: "full"`, `cert_name`, `cert_no`, `meta_info` |

- 通过后会将人员表 `face_verified = 1`、`face_verified_at` 更新。
- 人脸 1:N 逻辑在 `backend_dtcloud/digital_labor/services/aliyun_face_service.py`，控制器在 `controllers/person.py` 的 `_handle_face_verify_common`。

## 六、阿里云 API 与代码对应关系

本系统使用 **视觉智能开放平台 - 人脸人体（facebody 2019-12-30）**：

| 能力 | 接口 | 说明 |
|------|------|------|
| 创建人脸库 | CreateFaceDb | 启动时或首次录入时按需创建，库名由 `ALIYUN_FACE_DB_NAME` 配置 |
| 添加人脸样本 | AddFaceEntity | 每人一条，entity_id = person_id |
| 添加人脸数据 | AddFace（Advance） | 前端上传 base64，后端以 BytesIO 流调用 AddFaceAdvance |
| 人脸搜索 | SearchFace（Advance） | 核验时用拍照流调用 SearchFaceAdvance，按置信度判定 |

Endpoint：`facebody.cn-shanghai.aliyuncs.com`。SDK：`alibabacloud_facebody20191230`。

## 七、常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 一直走 mock | 未配置或未加载 .env | 确认 `.env` 在 `backend_dtcloud/`、变量名正确、重启服务 |
| 控制台报错“阿里云API调用失败” | Key 错误、未授权、网络或接口参数不符 | 检查 Key、RAM 授权、阿里云文档与代码是否一致 |
| 前端提示“验证失败” | 网络异常或阿里云返回不通过 | 看后端日志中阿里云返回的 Message/Reason |
| 希望临时关闭真人脸 | 生产需临时切回演示 | 设置 `FACE_VERIFY_ENABLED=false` 并重启 |

## 八、安全建议

1. **AccessKey** 仅在后端使用，不要写入前端或公开仓库
2. 使用 **RAM 子账号** + 最小权限，不要使用主账号 Key
3. 生产环境建议将 `.env` 设为 600 权限，并通过密钥管理服务或 CI 注入变量
