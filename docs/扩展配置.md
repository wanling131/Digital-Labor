# Digital Labor - 扩展配置

本文档说明可选的第三方服务配置。

---

## 一、阿里云人脸识别

### 1.1 功能说明

用于 H5 端人脸实名认证、活体检测。

| 状态 | 说明 |
|------|------|
| 已配置 | 调用阿里云真实接口 |
| 未配置 | 使用 mock 模式，返回通过 |

### 1.2 配置方式

```bash
# server/.env
ALIBABA_CLOUD_ACCESS_KEY_ID=你的AccessKeyId
ALIBABA_CLOUD_ACCESS_KEY_SECRET=你的AccessKeySecret

# 可选
ALIYUN_FACE_DB_NAME=digital_labor_face
ALIYUN_FACE_CONFIDENCE_THRESHOLD=60.48
```

### 1.3 阿里云准备

1. 开通 [人脸人体服务](https://vision.aliyun.com/facebody)
2. 创建 RAM 子账号，授权 `AliyunVIAPIFullAccess`
3. 获取 AccessKey ID 和 Secret

### 1.4 验证是否生效

```bash
curl -H "Authorization: Bearer <token>" http://localhost:3000/api/sys/feature-status
```

返回 `{"faceVerify": "aliyun"}` 表示已接入。

## 二、e签宝电子签章

### 2.1 功能说明

用于合同签署的法律效力存证。

| 状态 | 说明 |
|------|------|
| 已配置 | 调用 e签宝真实接口 |
| 未配置 | 使用本地 PDF 签署 |

### 2.2 配置方式

```bash
# server/.env
ESIGN_APP_ID=你的AppId
ESIGN_APP_SECRET=你的AppSecret
ESIGN_ENDPOINT=https://openapi.esign.cn
```

### 2.3 获取密钥

访问 [e签宝开放平台](https://open.esign.cn/) 注册并获取凭证。

## 三、Redis 缓存

### 3.1 功能说明

用于提升接口响应速度，支持多实例部署。

| 状态 | 说明 |
|------|------|
| 已配置 | 使用 Redis 缓存 |
| 未配置 | 使用内存缓存（单实例） |

### 3.2 配置方式

```bash
# server/.env
REDIS_URL=redis://localhost:6379/0
```

### 3.3 安装 Redis

```bash
# Ubuntu
sudo apt install redis-server
sudo systemctl start redis

# Docker
docker run -d -p 6379:6379 redis
```

## 四、短信服务

### 4.1 功能说明

用于发送验证码、通知短信。

| 状态 | 说明 |
|------|------|
| 已配置 | 可发送真实短信 |
| 未配置 | 短信功能不可用 |

### 4.2 配置方式

```bash
# server/.env
SMS_ACCESS_KEY_ID=你的AccessKeyId
SMS_ACCESS_KEY_SECRET=你的AccessKeySecret
SMS_SIGN_NAME=签名名称
SMS_TEMPLATE_CODE=模板Code
```

## 五、配置对照表

| 服务 | 环境变量 | 必需 | 不配置的影响 |
|------|----------|:----:|--------------|
| 数据库 | `DATABASE_URL` | ✅ | 无法启动 |
| JWT | `JWT_SECRET` | ⚠️ | 使用默认值（不安全） |
| 加密 | `ENCRYPTION_KEY` | ⚠️ | 敏感数据明文存储 |
| 人脸 | `ALIBABA_CLOUD_*` | ⭕ | mock 模式 |
| 电子签 | `ESIGN_*` | ⭕ | 本地 PDF 签署 |
| 缓存 | `REDIS_URL` | ⭕ | 内存缓存 |
| 短信 | `SMS_*` | ⭕ | 短信功能不可用 |

---

## 六、安全建议

1. **AccessKey** 仅在后端使用，不要写入前端或公开仓库
2. 使用 **RAM 子账号** + 最小权限
3. 生产环境 `.env` 设为 600 权限
4. 定期轮换密钥
