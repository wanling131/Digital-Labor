# Digital Labor - 部署指南

---

## 一、环境变量配置

### 1.1 必需配置

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `DATABASE_URL` | 数据库连接 | `sqlite:///./database/demo.sqlite3` 或 `postgresql+psycopg://...` |

### 1.2 安全配置（生产环境必须）

| 变量名 | 说明 | 生成命令 |
|--------|------|----------|
| `JWT_SECRET` | JWT 密钥 | `openssl rand -hex 32` |
| `ENCRYPTION_KEY` | 敏感数据加密密钥 | `openssl rand -hex 32` |

### 1.3 可选配置

| 变量名 | 说明 | 不配置的影响 |
|--------|------|--------------|
| `ALIBABA_CLOUD_ACCESS_KEY_ID` | 阿里云人脸 | 使用 mock 模式 |
| `ALIBABA_CLOUD_ACCESS_KEY_SECRET` | 阿里云人脸 | 使用 mock 模式 |
| `ESIGN_APP_ID` | e签宝电子签 | 使用本地 PDF 签署 |
| `REDIS_URL` | Redis 缓存 | 无缓存 |

### 1.4 配置示例

**开发环境（最简）**
```bash
# server/.env
DATABASE_URL=sqlite:///./database/demo.sqlite3
```

**生产环境**
```bash
# server/.env
ENV=production
DATABASE_URL=postgresql+psycopg://postgres:password@localhost:5432/digital_labor
JWT_SECRET=<openssl rand -hex 32 生成的值>
ENCRYPTION_KEY=<openssl rand -hex 32 生成的值>
CORS_ALLOW_ORIGINS=https://your-domain.com
```

## 三、生产环境部署

### 3.1 服务器要求

| 项目 | 最低要求 |
|------|----------|
| 操作系统 | Ubuntu 22.04 LTS |
| CPU | 2 核 |
| 内存 | 4GB |
| 存储 | 50GB |
| 端口 | 80, 443, 3000 |

### 3.2 安装依赖

```bash
# 系统依赖
sudo apt update && sudo apt install -y git nginx postgresql python3-pip nodejs

# 项目依赖
npm run install:all
```

### 3.3 配置数据库

```bash
sudo -u postgres psql
```
```sql
CREATE DATABASE digital_labor;
CREATE USER postgres WITH PASSWORD 'your-password';
GRANT ALL PRIVILEGES ON DATABASE digital_labor TO postgres;
```

### 3.4 配置环境变量

```bash
cp server/.env.example server/.env
nano server/.env
```

### 3.5 构建与启动

```bash
# 构建前端
npm run build

# 配置系统服务
sudo nano /etc/systemd/system/digital-labor.service
```

```ini
[Unit]
Description=Digital Labor Backend
After=network.target postgresql.service

[Service]
User=www-data
WorkingDirectory=/opt/digital-labor/server
ExecStart=/opt/digital-labor/server/.venv/bin/python -m digital_labor.run
EnvironmentFile=/opt/digital-labor/server/.env
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable digital-labor
sudo systemctl start digital-labor
```

### 3.6 Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /opt/digital-labor/web/out;
        try_files $uri $uri.html $uri/ =404;
    }

    location /api {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
    }
}
```

## 四、常见问题

### 后端启动失败

```bash
# 检查数据库连接
psql -h localhost -U postgres -d digital_labor

# 检查环境变量
cat server/.env

# 查看日志
sudo journalctl -u digital-labor -f
```

### 前端构建失败

```bash
# 检查 Node.js 版本
node -v  # 需要 >= 18

# 清理重装
cd web && rm -rf node_modules && npm install
```

## 七、系统嵌入（P2 集成）

### 7.1 环境变量

| 变量 | 说明 | 示例 |
|------|------|------|
| `NEXT_PUBLIC_LABOR_BASE_PATH` | 前端路由前缀 | `/labor` |
| `LABOR_API_PREFIX` | API 路由前缀 | `/labor` |
| `NEXT_PUBLIC_TENANT_ID` | 租户 ID | `tenant-001` |

### 7.2 嵌入方式

- **iframe**: 父应用通过 iframe 加载，postMessage 通信
- **微前端**: 作为子应用注册，props 注入配置
