## 二、本地演示部署

### 2.1 克隆项目

```bash
git clone https://github.com/wanling131/Digital-Labor.git
cd Digital-Labor
```

### 2.2 安装依赖

**一键安装（推荐）**
```bash
npm run install:all
```

**分步安装**
```bash
# 1. 根目录依赖
npm install

# 2. 前端依赖
cd web && npm install && cd ..

# 3. 后端依赖
cd server
python -m venv .venv

# Windows 激活
.\.venv\Scripts\activate

# macOS/Linux 激活
source .venv/bin/activate

pip install -r requirements-fastapi.txt
cd ..
```

### 2.3 配置环境变量

**最简配置（使用 SQLite）**

在 `server/.env` 文件中：
```bash
DATABASE_URL=sqlite:///./database/demo.sqlite3
```

### 2.4 启动服务

**方式一：同时启动前后端**
```bash
npm run dev
```

**方式二：分别启动**
```bash
# 终端1 - 启动后端
npm run dev:api

# 终端2 - 启动前端
npm run dev:web
```

### 2.5 访问系统

| 地址 | 说明 |
|------|------|
| http://localhost:3002 | 前端首页 |
| http://localhost:3002/login | 管理端登录 |
| http://localhost:3002/h5 | 工人端 |
| http://localhost:3000/api/health | 后端健康检查 |

### 2.6 默认账号

| 端 | 账号 | 密码 |
|----|------|------|
| 管理端 | admin | 123456 |
| 工人端 | 工号 + 姓名 | - |

### 2.7 初始化数据（可选）

如果需要演示数据：
```bash
npm run seed
```

## 三、服务器部署

### 3.1 服务器要求

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| 操作系统 | Ubuntu 20.04 | Ubuntu 22.04 LTS |
| CPU | 2 核 | 4 核 |
| 内存 | 4GB | 8GB |
| 存储 | 50GB | 100GB SSD |
| 带宽 | 5Mbps | 10Mbps |

### 3.2 安装系统依赖

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装基础工具
sudo apt install -y git curl wget build-essential

# 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 Python 3
sudo apt install -y python3 python3-pip python3-venv

# 安装 PostgreSQL
sudo apt install -y postgresql postgresql-contrib

# 安装 Nginx
sudo apt install -y nginx

# 安装 Redis（可选）
sudo apt install -y redis-server
```

### 3.3 配置 PostgreSQL

```bash
# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 创建数据库和用户
sudo -u postgres psql
```

```sql
CREATE DATABASE digital_labor;
CREATE USER labor_user WITH PASSWORD 'your-secure-password';
GRANT ALL PRIVILEGES ON DATABASE digital_labor TO labor_user;
ALTER DATABASE digital_labor OWNER TO labor_user;
\q
```

### 3.4 克隆项目

```bash
# 创建目录
sudo mkdir -p /opt/digital-labor
sudo chown -R $USER:$USER /opt/digital-labor

# 克隆代码
cd /opt
git clone https://github.com/wanling131/Digital-Labor.git digital-labor
cd digital-labor
```

### 3.5 安装项目依赖

```bash
# 根目录依赖
npm install

# 前端依赖
cd web && npm install && cd ..

# 后端依赖
cd server
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements-fastapi.txt
cd ..
```

### 3.6 配置环境变量

```bash
# 创建配置文件
cp server/.env.example server/.env
nano server/.env
```

**生产环境配置**
```bash
# ===== 基础配置 =====
ENV=production
PORT=3000
LOG_LEVEL=INFO

# ===== 数据库（必须） =====
DATABASE_URL=postgresql+psycopg://labor_user:your-secure-password@localhost:5432/digital_labor

# ===== 安全配置（必须） =====
# 生成命令: openssl rand -hex 32
JWT_SECRET=生成一个32位以上的随机字符串
ENCRYPTION_KEY=生成一个32位以上的随机字符串

# ===== 可选配置 =====
# Redis 缓存
REDIS_URL=redis://localhost:6379/0

# CORS（多个域名用逗号分隔）
CORS_ALLOW_ORIGINS=https://your-domain.com
```

### 3.7 构建前端

```bash
cd web
npm run build
cd ..
```

### 3.8 配置 Systemd 服务

```bash
sudo nano /etc/systemd/system/digital-labor.service
```

```ini
[Unit]
Description=Digital Labor Backend Service
After=network.target postgresql.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/digital-labor/server
Environment="PATH=/opt/digital-labor/server/.venv/bin"
ExecStart=/opt/digital-labor/server/.venv/bin/python -m digital_labor.run
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 设置目录权限
sudo chown -R www-data:www-data /opt/digital-labor

# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable digital-labor
sudo systemctl start digital-labor

# 检查状态
sudo systemctl status digital-labor
```

### 3.9 配置 Nginx

```bash
sudo nano /etc/nginx/sites-available/digital-labor
```

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /opt/digital-labor/web/.next;
        try_files $uri $uri.html $uri/ @next;
    }

    location @next {
        proxy_pass http://127.0.0.1:3002;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 后端 API
    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 文件上传大小限制
    client_max_body_size 50M;
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/digital-labor /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

### 3.10 配置 HTTPS（推荐）

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书
sudo certbot --nginx -d your-domain.com

# 自动续期测试
sudo certbot renew --dry-run
```

### 3.11 配置防火墙

```bash
# 启用防火墙
sudo ufw enable

# 开放端口
sudo ufw allow 22      # SSH
sudo ufw allow 80      # HTTP
sudo ufw allow 443     # HTTPS

# 查看状态
sudo ufw status
```

## 七、常见问题

### 7.1 后端启动失败

**检查数据库连接**
```bash
psql -h localhost -U labor_user -d digital_labor
```

**检查环境变量**
```bash
cat /opt/digital-labor/server/.env
```

**查看日志**
```bash
sudo journalctl -u digital-labor -n 50
```

### 7.2 前端访问空白

**检查构建**
```bash
ls -la /opt/digital-labor/web/.next
```

**检查 Nginx 配置**
```bash
sudo nginx -t
sudo tail -f /var/log/nginx/error.log
```

### 7.3 API 请求跨域

检查 `.env` 中的 `CORS_ALLOW_ORIGINS` 配置。

### 7.4 文件上传失败

检查 Nginx 配置中的 `client_max_body_size`。
