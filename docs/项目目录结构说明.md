## 八、开发规范

### 8.1 文件命名规范

- **组件文件**: PascalCase (如 `PageHeader.tsx`)
- **页面文件**: page.tsx (Next.js约定)
- **工具文件**: camelCase (如 `apiUtils.ts`)
- **样式文件**: kebab-case (如 `global-styles.css`)
- **常量文件**: UPPER_SNAKE_CASE (如 `API_CONSTANTS.ts`)

### 8.2 目录组织原则

1. **按功能模块组织**: 相关文件放在同一目录下
2. **公共组件抽离**: 通用组件放在 `components/ui/` 或 `components/common/`
3. **API集中管理**: 所有API请求封装在 `lib/api.ts`
4. **类型定义集中**: 共享类型定义放在 `types/` 目录

### 8.3 代码规范

- 使用 TypeScript 进行类型检查
- 使用 ESLint 进行代码规范检查
- 使用 Prettier 进行代码格式化
- 组件使用函数式组件 + Hooks
- API 使用 RESTful 风格
