# 环境变量示例

部署或本地运行时，可创建 **`.env`** 文件（项目根目录或 server 目录），参考下列内容。`.env` 已加入 .gitignore，不会提交。

```bash
# 后端服务端口（默认 3000）
PORT=3000

# JWT 密钥（生产环境务必修改）
JWT_SECRET=labor-secret-change-in-prod

# 运行环境：development | production
NODE_ENV=development
```

说明：
- **PORT**：后端 API 监听端口。
- **JWT_SECRET**：登录签发的 JWT 使用该密钥，生产必须改为强随机字符串。
- **NODE_ENV**：部分库会据此切换行为；生产建议设为 production。

前端构建时若需指定 API 基地址，在 web 项目中使用 `NEXT_PUBLIC_API_BASE`（Next.js）。

## 人脸 / 电子签第三方配置

### 阿里云人脸验证

| 变量名 | 说明 | 必填 |
|--------|------|:----:|
| `ALIYUN_ACCESS_KEY_ID` | 阿里云 AccessKey ID | ✅ |
| `ALIYUN_ACCESS_KEY_SECRET` | 阿里云 AccessKey Secret | ✅ |
| `FACE_VERIFY_ENABLED` | 是否启用人脸核验（true/false） | 否，默认true |

### e签宝电子签章

| 变量名 | 说明 | 必填 |
|--------|------|:----:|
| `ESIGN_APP_ID` | e签宝 AppId | ✅ |
| `ESIGN_APP_SECRET` | e签宝 App Secret | ✅ |
| `ESIGN_ENDPOINT` | e签宝 API 端点 | 否，默认https://openapi.esign.cn |

### 配置示例

```bash
# 阿里云人脸验证
ALIYUN_ACCESS_KEY_ID=your_access_key_id
ALIYUN_ACCESS_KEY_SECRET=your_access_key_secret
FACE_VERIFY_ENABLED=true

# e签宝电子签章
ESIGN_APP_ID=your_esign_app_id
ESIGN_APP_SECRET=your_esign_app_secret
ESIGN_ENDPOINT=https://openapi.esign.cn
```

**注意**：
- 开发环境未配置阿里云密钥时，系统会自动使用模拟服务（mock）
- 生产环境务必配置真实的阿里云密钥，否则人脸验证功能将不可用
- 密钥信息请妥善保管，不要提交到代码仓库

## 日志监控（可选）

接入 Sentry、健康检查告警等时使用，详见 **`docs/日志监控接入方案.md`**。

| 变量名 | 说明 |
|--------|------|
| `SENTRY_DSN` | Sentry 后端 DSN，不设则不上报 |
| `NEXT_PUBLIC_SENTRY_DSN` / `SENTRY_DSN` | 前端（web）Sentry DSN |
| `HEALTH_URL` | 健康检查地址（自建告警脚本用） |
| `DING_WEBHOOK` / `DING_SECRET` | 钉钉机器人 Webhook 与加签密钥（告警脚本用） |
