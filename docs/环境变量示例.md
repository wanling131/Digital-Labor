# Digital Labor · 环境变量示例

> 本文档说明所有可用的环境变量配置。文档索引见 [docs/README.md](./README.md)。

---

## 一、后端环境变量

### 1.1 基础配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `PORT` | 3000 | 服务监听端口 |
| `JWT_SECRET` | labor-secret-change-in-prod | JWT签名密钥 |
| `TOKEN_VERSION` | 1 | token 版本（用于强制失效旧 token） |
| `ENCRYPTION_KEY` | (空) | 敏感字段加密密钥（与旧系统兼容） |
| `DATABASE_URL` | 见下 | PostgreSQL 连接串（SQLAlchemy URL） |

### 1.2 第三方服务

#### 阿里云人脸验证

| 变量名 | 必填 | 说明 |
|--------|:----:|------|
| `ALIYUN_ACCESS_KEY_ID` | ✅ | 阿里云AccessKey ID |
| `ALIYUN_ACCESS_KEY_SECRET` | ✅ | 阿里云AccessKey Secret |
| `FACE_VERIFY_ENABLED` | 否 | 是否启用人脸核验，默认true |

#### e签宝电子签章

| 变量名 | 必填 | 说明 |
|--------|:----:|------|
| `ESIGN_APP_ID` | ✅ | e签宝AppId |
| `ESIGN_APP_SECRET` | ✅ | e签宝App Secret |
| `ESIGN_ENDPOINT` | 否 | API端点，默认https://openapi.esign.cn |

### 1.3 日志监控

| 变量名 | 说明 |
|--------|------|
| `SENTRY_DSN` | Sentry错误监控DSN |
| `LOG_LEVEL` | 日志级别: debug/info/warn/error |

## 三、配置示例

### 3.1 开发环境

```bash
# backend_dtcloud/.env
PORT=3000
JWT_SECRET=dev-secret-key
TOKEN_VERSION=1
ENCRYPTION_KEY=change-me
DATABASE_URL=postgresql+psycopg://postgres:postgres@localhost:5432/digital_labor

# 开发环境可不配置第三方服务，使用mock
```

### 3.2 生产环境

```bash
# backend_dtcloud/.env
PORT=3000
JWT_SECRET=your-strong-random-secret-key-at-least-32-chars
TOKEN_VERSION=1
ENCRYPTION_KEY=your-strong-encryption-key
DATABASE_URL=postgresql+psycopg://user:pass@db-host:5432/digital_labor

# 阿里云人脸验证
ALIYUN_ACCESS_KEY_ID=your_access_key_id
ALIYUN_ACCESS_KEY_SECRET=your_access_key_secret
FACE_VERIFY_ENABLED=true

# e签宝电子签章
ESIGN_APP_ID=your_esign_app_id
ESIGN_APP_SECRET=your_esign_app_secret
ESIGN_ENDPOINT=https://openapi.esign.cn

# 日志监控
SENTRY_DSN=https://xxx@xxx.ingest.sentry.io/xxx
LOG_LEVEL=info
```

### 3.3 前端构建

```bash
# 前端工程（Vite）示例：.env.production
VITE_API_BASE=https://api.your-domain.com
VITE_SENTRY_DSN=https://xxx@xxx.ingest.sentry.io/xxx
```

## 四、安全注意事项

1. **JWT_SECRET**: 生产环境必须使用强随机字符串，至少32位
2. **密钥保管**: 不要将真实密钥提交到代码仓库
3. **文件权限**: `.env`文件权限设置为600
4. **定期更换**: 建议定期更换JWT_SECRET和第三方密钥

## 五、验证配置

```bash
# 检查环境变量是否生效
cd backend_dtcloud
python -c "import os; print(os.getenv('DATABASE_URL'))"

# 启动服务查看日志确认
python -m digital_labor.run
```

**人脸/活体**：配置阿里云密钥后，可请求 `GET /api/sys/feature-status`，响应中 `faceVerify` 为 `aliyun` 表示已接入真实服务，为 `mock` 表示使用本地模拟。详见 [人脸活体配置说明.md](./人脸活体配置说明.md)。
