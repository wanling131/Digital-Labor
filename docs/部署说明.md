## 三、生产环境配置

### 3.1 后端服务

当前主线后端为 **Python（FastAPI 适配层）+ PostgreSQL**，代码位于 `backend_dtcloud/`。

**环境变量**（推荐在 `backend_dtcloud/.env` 配置）：

```bash
# 服务配置
PORT=3000

# 安全密钥（必须修改！）
JWT_SECRET=your-strong-random-secret-key-here
TOKEN_VERSION=1
ENCRYPTION_KEY=change-me

# PostgreSQL（示例：本机）
DATABASE_URL=postgresql+psycopg://postgres:postgres@localhost:5432/digital_labor

# 可选：第三方服务
# 人脸/活体：配置后使用阿里云实人认证；不配置或 FACE_VERIFY_ENABLED=false 时使用本地 mock
ALIYUN_ACCESS_KEY_ID=your_key
ALIYUN_ACCESS_KEY_SECRET=your_secret
ESIGN_APP_ID=your_app_id
ESIGN_APP_SECRET=your_secret
```

**启动方式**:

```bash
cd backend_dtcloud
python -m venv .venv
. .venv/bin/activate  # Windows: .\\.venv\\Scripts\\activate
pip install -r requirements-fastapi.txt

# 方式1：直接启动（FastAPI）
python -m digital_labor.run

# 方式2：systemd（示例）
# ExecStart=/path/to/backend_dtcloud/.venv/bin/python -m digital_labor.run
```

### 3.2 前端部署
当前主线前端为 **Vue 3.3.4**（如需部署请以对应前端工程的构建产物为准）。\
若你仍在使用仓库内 `web/`（历史实现），可继续按其工程 README/脚本构建部署；后端 `/api` 反代规则不变。

### 3.3 Nginx 配置（推荐）

建议 Nginx：前端静态资源（或前端服务）+ 后端 API 反代分离。

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端（示例：静态站点）
    location / {
        root /path/to/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理到后端（FastAPI，端口 3000）
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # uploads 静态资源（与旧版兼容：/uploads -> server/uploads）
    location /uploads/ {
        alias /path/to/Digital-Labor/server/uploads/;
        autoindex off;
        expires 7d;
    }
}
```

## 四、数据库备份

### 4.1 自动备份脚本

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/path/to/backups"
DATE=$(date +%Y%m%d_%H%M%S)

# PostgreSQL 备份（按你的连接信息调整）
pg_dump -h 127.0.0.1 -p 5432 -U postgres -d digital_labor > "$BACKUP_DIR/digital_labor_$DATE.sql"

# 保留最近30天备份
find "$BACKUP_DIR" -name "digital_labor_*.sql" -mtime +30 -delete

echo "Backup completed: digital_labor_$DATE.sql"
```

### 4.2 定时任务

```bash
# 每天凌晨2点备份
0 2 * * * /path/to/backup.sh >> /var/log/labor-backup.log 2>&1
```

## 六、常见问题

### 6.1 文件权限

```bash
# 确保数据库目录可写
# 仅针对 uploads 目录示例（按实际部署路径调整）
chmod 755 /path/to/Digital-Labor/server/uploads
chown -R www-data:www-data /path/to/Digital-Labor/server/uploads
```

### 6.2 跨域问题

生产环境使用Nginx反向代理，无需额外CORS配置。

```bash
git pull
```

