## 三、生产环境配置

### 3.1 后端服务

**环境变量** (`server/.env`):

```bash
# 服务配置
PORT=3000
NODE_ENV=production

# 安全密钥（必须修改！）
JWT_SECRET=your-strong-random-secret-key-here

# 可选：第三方服务
# 人脸/活体：配置后使用阿里云实人认证；不配置或 FACE_VERIFY_ENABLED=false 时使用本地 mock
ALIYUN_ACCESS_KEY_ID=your_key
ALIYUN_ACCESS_KEY_SECRET=your_secret
ESIGN_APP_ID=your_app_id
ESIGN_APP_SECRET=your_secret
```

**启动方式**:

```bash
cd server
npm install --production

# 方式1: 直接启动
npm run start

# 方式2: PM2守护进程
pm2 start server.js -n labor-api
pm2 save
pm2 startup
```

### 3.2 前端部署

Next.js 默认构建产出为 `.next`，需用 **Node 进程** 运行 `next start`，再由 Nginx 反代。**不要**使用不存在的 `web/dist`。

```bash
cd web
npm install
npm run build

# 使用 PM2 启动前端（端口 3001），与后端 3000 分离
pm2 start npm --name labor-web -p 3001 -- start
# 或直接：npm run start（前台运行，端口 3001）
```

**可选：纯静态导出**（无需 Node 跑前端时）：在 `web/next.config.mjs` 中增加 `output: 'export'`，再执行 `npm run build`，将生成 `web/out/` 目录；Nginx 的 `root` 指向 `web/out`，并单独配置 `/api/` 反代到后端。注意静态导出后开发时的 rewrites 不再生效，生产依赖 Nginx 做 `/api` 反代。

### 3.3 Nginx配置（推荐：Node 模式）

前端由 Node 提供（`next start -p 3001`），Nginx 只做反向代理：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端（Next 服务，端口 3001）
    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # API 反向代理到后端（端口 3000）
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

若采用**纯静态导出**（`output: 'export'`，部署 `web/out/`），则使用：

```nginx
    root /path/to/web/out;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
    location /api/ {
        # 同上，proxy_pass http://127.0.0.1:3000;
    }
```

## 四、数据库备份

### 4.1 自动备份脚本

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/path/to/backups"
DB_FILE="/path/to/server/data/labor.db"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份
cp "$DB_FILE" "$BACKUP_DIR/labor_$DATE.db"

# 保留最近30天备份
find "$BACKUP_DIR" -name "labor_*.db" -mtime +30 -delete

echo "Backup completed: labor_$DATE.db"
```

### 4.2 定时任务

```bash
# 每天凌晨2点备份
0 2 * * * /path/to/backup.sh >> /var/log/labor-backup.log 2>&1
```

## 六、常见问题

### 6.1 数据库锁定

SQLite并发写入可能锁定，建议：
- 避免高并发写入
- 考虑迁移到PostgreSQL/MySQL

### 6.2 文件权限

```bash
# 确保数据库目录可写
chmod 755 server/data
chown -R www-data:www-data server/data
```

### 6.3 跨域问题

生产环境使用Nginx反向代理，无需额外CORS配置。
