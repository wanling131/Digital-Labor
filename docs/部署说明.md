# Digital Labor · 部署说明

本文档说明前后端分离下的**开发**、**构建**与**生产部署**方式，便于运维与后期维护。

---

## 一、前后端分离说明

| 部分 | 技术 | 开发地址 | 生产部署方式 |
|------|------|----------|--------------|
| 前端 | Vite + React | 独立 dev server（如 5173） | 构建为静态资源，由 Nginx/OSS/CDN 托管 |
| 后端 | Express | 独立 API（如 3000） | Node 进程或 Docker，仅提供 `/api/*` |

- **开发时**：前端通过 Vite 代理将 `/api` 转发到后端，避免跨域。
- **生产时**：前端与后端可部署在不同域名/机器，前端配置 `VITE_API_BASE` 或 Nginx 反向代理到后端 API。

---

## 二、开发环境

```bash
# 1. 安装依赖（根目录 + server + web）
npm run install:all

# 2. 同时启动后端 + 前端
npm run dev
```

- 仅启动后端：`npm run dev:server`（API: http://localhost:3000）
- 仅启动前端：`npm run dev:web`（需先有 web 目录且已安装依赖，前端: http://localhost:5173）

前端开发时需保证后端已运行或配置好 API 代理目标。

---

## 三、构建

```bash
# 构建前端（产出在 web/dist）
npm run build
# 或
npm run build:web
```

后端无需构建，直接使用 Node 运行 `server/`。

---

## 四、生产部署

### 4.1 后端

1. 在服务器上进入 `server` 目录，执行 `npm install --production`（或全局只安装 production 依赖）。
2. 配置环境变量（建议使用 `.env` 或系统环境变量）：
   - `PORT`：监听端口，默认 3000
   - `JWT_SECRET`：务必改为强随机字符串
   - `NODE_ENV=production`
3. 启动方式二选一：
   - **直接启动**：`npm run start` 或 `node server.js`
   - **进程守护**：使用 pm2：`pm2 start server.js -n labor-api --cwd server`

数据库文件默认落在 `server/data/labor.db`，请保证该目录可写并做好备份。

### 4.2 前端

1. 在构建机或本机执行 `npm run build:web`，得到 `web/dist`。
2. 将 `web/dist` 内所有文件部署到：
   - **Nginx**：配置 root 指向 `web/dist`，并配置 `/api` 反向代理到后端地址；或
   - **静态托管**（OSS、Vercel、Netlify 等）：若前后端同域则需该平台支持代理；否则前端需配置 API 基础地址（如 `VITE_API_BASE=https://api.xxx.com`），构建时写入。
3. 若使用 Nginx，示例：

```nginx
server {
  listen 80;
  server_name your-domain.com;
  root /path/to/web/dist;
  index index.html;
  location / {
    try_files $uri $uri/ /index.html;
  }
  location /api/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

### 4.3 前后端同机部署（可选）

- 后端监听 3000，Nginx 监听 80，静态资源 root 指向 `web/dist`，`/api` 代理到 3000（如上）。
- 或后端与 Nginx 同机，前端仅通过 Nginx 提供，API 通过 Nginx 转发。

## 五、环境变量汇总

| 变量 | 说明 | 默认 |
|------|------|------|
| PORT | 后端监听端口 | 3000 |
| JWT_SECRET | JWT 签名密钥 | labor-secret-change-in-prod |
| NODE_ENV | development / production | development |

前端构建时若需指定 API 地址，使用 Vite 环境变量（如 `VITE_API_BASE`），在 `web` 内配置并在请求基地址中使用。

---

## 六、数据与备份

- SQLite 数据文件：`server/data/labor.db`
- 上传文件（如合同模板）：`server/uploads/`
- 建议定期备份 `server/data/` 与 `server/uploads/`。
