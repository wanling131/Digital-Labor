# Digital Labor 服务器部署文档

## 1. 环境准备

### 1.1 服务器要求
- **操作系统**：Ubuntu 22.04 LTS（推荐）
- **CPU**：至少 2 核
- **内存**：至少 4GB
- **存储空间**：至少 50GB
- **网络**：支持公网访问，开放必要端口（80、443、3000）

### 1.2 系统更新
```bash
sudo apt update && sudo apt upgrade -y
```

### 1.3 安装必要依赖
```bash
sudo apt install -y git curl wget build-essential nginx postgresql postgresql-contrib python3 python3-pip python3-venv
```

## 2. 数据库配置

### 2.1 PostgreSQL 配置
1. 启动 PostgreSQL 服务
```bash
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

2. 创建数据库和用户
```bash
sudo -u postgres psql
```
```sql
CREATE DATABASE digital_labor;
CREATE USER postgres WITH PASSWORD 'postgres';
GRANT ALL PRIVILEGES ON DATABASE digital_labor TO postgres;
\q
```

### 2.2 测试数据库连接
```bash
psql -h localhost -U postgres -d digital_labor
```

## 3. 后端部署

### 3.1 克隆代码
```bash
cd /opt
sudo git clone https://your-repo-url/digital-labor.git
sudo chown -R $USER:$USER digital-labor
cd digital-labor
```

### 3.2 配置虚拟环境
```bash
cd server
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3.3 环境变量配置
创建 `.env` 文件：
```bash
nano .env
```
添加以下内容：
```
DATABASE_URL=postgresql+psycopg://postgres:postgres@localhost:5432/digital_labor
JWT_SECRET=your-secret-key-here
PORT=3000
```

### 3.4 初始化数据库
```bash
cd ../database
python init_and_import.py --with-seed
```

### 3.5 配置系统服务
创建服务文件：
```bash
sudo nano /etc/systemd/system/digital-labor-backend.service
```
添加以下内容：
```
[Unit]
Description=Digital Labor Backend Service
After=network.target postgresql.service

[Service]
User=your-username
WorkingDirectory=/opt/digital-labor/server
ExecStart=/opt/digital-labor/server/.venv/bin/python -m digital_labor.run
EnvironmentFile=/opt/digital-labor/server/.env
Restart=always
RestartSec=30

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start digital-labor-backend
sudo systemctl enable digital-labor-backend
```

### 3.6 验证后端服务
```bash
curl http://localhost:3000/api/health
```
预期输出：
```json
{"ok": true}
```

## 4. 前端部署

### 4.1 安装 Node.js
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
```

### 4.2 安装依赖并构建
```bash
cd /opt/digital-labor/web
npm install
npm run build
```

### 4.3 配置系统服务
创建服务文件：
```bash
sudo nano /etc/systemd/system/digital-labor-frontend.service
```
添加以下内容：
```
[Unit]
Description=Digital Labor Frontend Service
After=network.target digital-labor-backend.service

[Service]
User=your-username
WorkingDirectory=/opt/digital-labor/web
ExecStart=/usr/bin/npm start
Environment=PORT=3001
Restart=always
RestartSec=30

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start digital-labor-frontend
sudo systemctl enable digital-labor-frontend
```

### 4.4 验证前端服务
```bash
curl http://localhost:3001
```
预期输出：前端 HTML 内容

## 5. Nginx 配置

### 5.1 配置 Nginx 反向代理
创建配置文件：
```bash
sudo nano /etc/nginx/sites-available/digital-labor
```
添加以下内容：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/digital-labor /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 5.2 配置 HTTPS（可选）
```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 7. 监控与维护

### 7.1 查看服务状态
```bash
sudo systemctl status digital-labor-backend
sudo systemctl status digital-labor-frontend
```

### 7.2 查看日志
```bash
sudo journalctl -u digital-labor-backend -f
sudo journalctl -u digital-labor-frontend -f
```

### 7.3 数据库备份
创建备份脚本：
```bash
nano /opt/digital-labor/scripts/backup-db.sh
```
添加以下内容：
```bash
#!/bin/bash
BACKUP_DIR="/opt/digital-labor/backups"
DATE=$(date +"%Y-%m-%d_%H-%M-%S")
mkdir -p $BACKUP_DIR
pg_dump -h localhost -U postgres -d digital_labor > $BACKUP_DIR/digital_labor_$DATE.sql
echo "Backup completed: $BACKUP_DIR/digital_labor_$DATE.sql"
```

设置定时备份：
```bash
sudo chmod +x /opt/digital-labor/scripts/backup-db.sh
sudo crontab -e
```
添加以下内容：
```
0 0 * * * /opt/digital-labor/scripts/backup-db.sh
```

## 8. 常见问题排查

### 8.1 后端服务启动失败
- 检查数据库连接：`psql -h localhost -U postgres -d digital_labor`
- 检查环境变量：`cat /opt/digital-labor/server/.env`
- 查看日志：`sudo journalctl -u digital-labor-backend -f`

### 8.2 前端服务启动失败
- 检查 Node.js 版本：`node -v`
- 检查构建是否成功：`ls -la /opt/digital-labor/web/.next`
- 查看日志：`sudo journalctl -u digital-labor-frontend -f`

### 8.3 Nginx 配置问题
- 检查配置文件：`sudo nginx -t`
- 查看 Nginx 日志：`sudo tail -f /var/log/nginx/error.log`

### 8.4 数据库连接问题
- 检查 PostgreSQL 服务状态：`sudo systemctl status postgresql`
- 检查数据库用户权限：`sudo -u postgres psql -c "SELECT * FROM pg_user;"`

## 11. 附录

### 11.1 环境变量说明

| 变量名 | 说明 | 默认值 |
|-------|------|-------|
| DATABASE_URL | 数据库连接字符串 | postgresql+psycopg://postgres:postgres@localhost:5432/digital_labor |
| JWT_SECRET | JWT 密钥（生产环境必须修改！） | 随机生成的字符串 |
| ENCRYPTION_KEY | 敏感数据加密密钥 | 32位随机字符串 |
| PORT | 后端服务端口 | 3000 |
| DEBUG | 调试模式 | False |

**⚠️ 安全提醒：**
- **JWT_SECRET** 和 **ENCRYPTION_KEY** 在生产环境必须使用强随机字符串
- 生成命令：`openssl rand -hex 32`
- **切勿**将真实密钥提交到 Git 仓库
- 如需配置第三方服务（阿里云人脸、e签宝等），请参考 `docs/环境变量示例.md`

### 11.2 服务管理命令
- 启动服务：`sudo systemctl start digital-labor-backend digital-labor-frontend`
- 停止服务：`sudo systemctl stop digital-labor-backend digital-labor-frontend`
- 重启服务：`sudo systemctl restart digital-labor-backend digital-labor-frontend`
- 查看状态：`sudo systemctl status digital-labor-backend digital-labor-frontend`

### 11.3 紧急恢复
1. 停止服务：`sudo systemctl stop digital-labor-backend digital-labor-frontend`
2. 恢复数据库：`psql -h localhost -U postgres -d digital_labor < /opt/digital-labor/backups/latest-backup.sql`
3. 启动服务：`sudo systemctl start digital-labor-backend digital-labor-frontend`

## 12. DTCloud 官方一键部署（推荐）

DTCloud（中亿丰数字科技开发的基于 BIM+AIoT 的低代码/无代码开发平台）在服务器上的部署推荐使用官方提供的一键安装脚本，这是最简便且不易出错的方式。

### 12.1 前置准备

**服务器要求：**
- 操作系统：推荐 Ubuntu 20.04 或 Ubuntu 22.04（官方脚本主要针对 Ubuntu 优化）
- 配置建议：建议至少 4核 CPU / 8GB 内存（运行 BIM 和 AIoT 服务较消耗资源），硬盘建议 50GB 以上
- 网络：确保服务器可以访问外网（用于下载依赖包），并开放相应端口（默认 Web 端口通常为 80/8080）
- 用户权限：请使用 root 用户或具有 sudo 权限的用户登录
- 环境清理：如果是新服务器最好；如果旧服务器安装过 PostgreSQL 或其他冲突服务，建议先清理

### 12.2 一键安装步骤

**第一步：下载安装脚本**

```bash
# 创建安装目录
mkdir -p /opt/DTCloud_Install
cd /opt/DTCloud_Install

# 克隆官方安装脚本仓库（使用 Gitee）
git clone https://gitee.com/dtcloud360/dtcloud-install.git

# 进入脚本目录
cd dtcloud-install
```

**第二步：执行安装脚本**

```bash
# 赋予执行权限
chmod +x dtcloud2.0.sh

# 执行脚本
bash dtcloud2.0.sh
```

**交互过程说明：**
- 脚本启动后，会提示输入服务器公网 IP 地址
- 脚本会自动检测系统环境，安装必要的依赖（Python, PostgreSQL, Node.js 等）
- 自动初始化数据库和配置文件
- 等待进度条走完，显示 "Installation Complete" 或类似成功提示

### 12.3 验证与访问

安装完成后，脚本通常会输出访问地址、默认账号和密码：
- 访问地址：`http://<服务器IP>:端口`（例如 `http://1.2.3.4:8080` 或直接 `http://1.2.3.4`）
- 默认账号：常见为 admin
- 默认密码：脚本输出中会显示，或者尝试 123456 / admin123

### 12.4 常用运维命令

```bash
# 查看状态（可能命令示例，具体视脚本生成情况）
systemctl status dtcloud

# 或在安装目录下运行
./dtcloud2.0.sh status

# 重启服务
./dtcloud2.0.sh restart

# 停止服务
./dtcloud2.0.sh stop
```

### 12.5 注意事项

**系统兼容性：**
- 该一键脚本主要支持 Ubuntu
- CentOS 用户：可能需要手动安装依赖，然后参考源码部署方式

**防火墙/安全组：**
- 如果在云服务器（阿里云、腾讯云、华为云等）上部署，务必去云控制台的安全组设置中，放行 Web 访问端口（如 80, 8080, 3000 等）

**中文路径问题：**
- 确保服务器用户名和安装路径不包含中文字符，否则 Python 或 PostgreSQL 可能会出现编码错误

**手动部署（进阶）：**
- 如果一键脚本失败，可以选择手动部署
- 手动安装 PostgreSQL（创建数据库和用户）
- 手动安装 Python 环境并 pip install 依赖
- 从 Gitee 拉取 dtcloud 主代码库
- 修改 dtcloud.conf 配置文件，填入数据库账号密码
- 手动启动服务
