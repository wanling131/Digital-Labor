# Digital Labor 服务器部署文档

## 1. 环境准备

### 1.1 服务器要求
- **操作系统**：Ubuntu 22.04 LTS（推荐）
- **CPU**：至少 2 核
- **内存**：至少 4GB
- **存储空间**：至少 50GB
- **网络**：支持公网访问，开放必要端口（80、443、3000）

### 1.2 系统更新
```bash
sudo apt update && sudo apt upgrade -y
```

### 1.3 安装必要依赖
```bash
sudo apt install -y git curl wget build-essential nginx postgresql postgresql-contrib python3 python3-pip python3-venv
```

## 2. 数据库配置

### 2.1 PostgreSQL 配置
1. 启动 PostgreSQL 服务
```bash
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

2. 创建数据库和用户
```bash
sudo -u postgres psql
```
```sql
CREATE DATABASE digital_labor;
CREATE USER postgres WITH PASSWORD 'postgres';
GRANT ALL PRIVILEGES ON DATABASE digital_labor TO postgres;
\q
```

### 2.2 测试数据库连接
```bash
psql -h localhost -U postgres -d digital_labor
```

## 3. 后端部署

### 3.1 克隆代码
```bash
cd /opt
sudo git clone https://your-repo-url/digital-labor.git
sudo chown -R $USER:$USER digital-labor
cd digital-labor
```

### 3.2 配置虚拟环境
```bash
cd backend_dtcloud
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3.3 环境变量配置
创建 `.env` 文件：
```bash
nano .env
```
添加以下内容：
```
DATABASE_URL=postgresql+psycopg://postgres:postgres@localhost:5432/digital_labor
JWT_SECRET=your-secret-key-here
PORT=3000
```

### 3.4 初始化数据库
```bash
cd ../database
python init_and_import.py --with-seed
```

### 3.5 配置系统服务
创建服务文件：
```bash
sudo nano /etc/systemd/system/digital-labor-backend.service
```
添加以下内容：
```
[Unit]
Description=Digital Labor Backend Service
After=network.target postgresql.service

[Service]
User=your-username
WorkingDirectory=/opt/digital-labor/backend_dtcloud
ExecStart=/opt/digital-labor/backend_dtcloud/.venv/bin/python -m digital_labor.run
EnvironmentFile=/opt/digital-labor/backend_dtcloud/.env
Restart=always
RestartSec=30

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start digital-labor-backend
sudo systemctl enable digital-labor-backend
```

### 3.6 验证后端服务
```bash
curl http://localhost:3000/api/health
```
预期输出：
```json
{"ok": true}
```

## 4. 前端部署

### 4.1 安装 Node.js
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
```

### 4.2 安装依赖并构建
```bash
cd /opt/digital-labor/web
npm install
npm run build
```

### 4.3 配置系统服务
创建服务文件：
```bash
sudo nano /etc/systemd/system/digital-labor-frontend.service
```
添加以下内容：
```
[Unit]
Description=Digital Labor Frontend Service
After=network.target digital-labor-backend.service

[Service]
User=your-username
WorkingDirectory=/opt/digital-labor/web
ExecStart=/usr/bin/npm start
Environment=PORT=3001
Restart=always
RestartSec=30

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start digital-labor-frontend
sudo systemctl enable digital-labor-frontend
```

### 4.4 验证前端服务
```bash
curl http://localhost:3001
```
预期输出：前端 HTML 内容

## 5. Nginx 配置

### 5.1 配置 Nginx 反向代理
创建配置文件：
```bash
sudo nano /etc/nginx/sites-available/digital-labor
```
添加以下内容：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/digital-labor /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 5.2 配置 HTTPS（可选）
```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 7. 监控与维护

### 7.1 查看服务状态
```bash
sudo systemctl status digital-labor-backend
sudo systemctl status digital-labor-frontend
```

### 7.2 查看日志
```bash
sudo journalctl -u digital-labor-backend -f
sudo journalctl -u digital-labor-frontend -f
```

### 7.3 数据库备份
创建备份脚本：
```bash
nano /opt/digital-labor/scripts/backup-db.sh
```
添加以下内容：
```bash
#!/bin/bash
BACKUP_DIR="/opt/digital-labor/backups"
DATE=$(date +"%Y-%m-%d_%H-%M-%S")
mkdir -p $BACKUP_DIR
pg_dump -h localhost -U postgres -d digital_labor > $BACKUP_DIR/digital_labor_$DATE.sql
echo "Backup completed: $BACKUP_DIR/digital_labor_$DATE.sql"
```

设置定时备份：
```bash
sudo chmod +x /opt/digital-labor/scripts/backup-db.sh
sudo crontab -e
```
添加以下内容：
```
0 0 * * * /opt/digital-labor/scripts/backup-db.sh
```

## 8. 常见问题排查

### 8.1 后端服务启动失败
- 检查数据库连接：`psql -h localhost -U postgres -d digital_labor`
- 检查环境变量：`cat /opt/digital-labor/backend_dtcloud/.env`
- 查看日志：`sudo journalctl -u digital-labor-backend -f`

### 8.2 前端服务启动失败
- 检查 Node.js 版本：`node -v`
- 检查构建是否成功：`ls -la /opt/digital-labor/web/.next`
- 查看日志：`sudo journalctl -u digital-labor-frontend -f`

### 8.3 Nginx 配置问题
- 检查配置文件：`sudo nginx -t`
- 查看 Nginx 日志：`sudo tail -f /var/log/nginx/error.log`

### 8.4 数据库连接问题
- 检查 PostgreSQL 服务状态：`sudo systemctl status postgresql`
- 检查数据库用户权限：`sudo -u postgres psql -c "SELECT * FROM pg_user;"`

## 11. 附录

### 11.1 环境变量说明
| 变量名 | 说明 | 默认值 |
|-------|------|-------|
| DATABASE_URL | 数据库连接字符串 | postgresql+psycopg://postgres:postgres@localhost:5432/digital_labor |
| JWT_SECRET | JWT 密钥 | 随机生成的字符串 |
| PORT | 后端服务端口 | 3000 |
| DEBUG | 调试模式 | False |

### 11.2 服务管理命令
- 启动服务：`sudo systemctl start digital-labor-backend digital-labor-frontend`
- 停止服务：`sudo systemctl stop digital-labor-backend digital-labor-frontend`
- 重启服务：`sudo systemctl restart digital-labor-backend digital-labor-frontend`
- 查看状态：`sudo systemctl status digital-labor-backend digital-labor-frontend`

### 11.3 紧急恢复
1. 停止服务：`sudo systemctl stop digital-labor-backend digital-labor-frontend`
2. 恢复数据库：`psql -h localhost -U postgres -d digital_labor < /opt/digital-labor/backups/latest-backup.sql`
3. 启动服务：`sudo systemctl start digital-labor-backend digital-labor-frontend`
