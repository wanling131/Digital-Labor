## 三、生产环境配置

### 3.1 后端服务

**环境变量** (`server/.env`):

```bash
# 服务配置
PORT=3000
NODE_ENV=production

# 安全密钥（必须修改！）
JWT_SECRET=your-strong-random-secret-key-here

# 可选：第三方服务
ALIYUN_ACCESS_KEY_ID=your_key
ALIYUN_ACCESS_KEY_SECRET=your_secret
ESIGN_APP_ID=your_app_id
ESIGN_APP_SECRET=your_secret
```

**启动方式**:

```bash
cd server
npm install --production

# 方式1: 直接启动
npm run start

# 方式2: PM2守护进程
pm2 start server.js -n labor-api
pm2 save
pm2 startup
```

### 3.2 前端部署

```bash
# 构建生产包
cd web
npm install
npm run build

# 部署web/dist到Nginx
```

### 3.3 Nginx配置

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 前端静态资源
    root /path/to/web/dist;
    index index.html;
    
    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
    
    # 静态文件缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

## 四、数据库备份

### 4.1 自动备份脚本

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/path/to/backups"
DB_FILE="/path/to/server/data/labor.db"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份
cp "$DB_FILE" "$BACKUP_DIR/labor_$DATE.db"

# 保留最近30天备份
find "$BACKUP_DIR" -name "labor_*.db" -mtime +30 -delete

echo "Backup completed: labor_$DATE.db"
```

### 4.2 定时任务

```bash
# 每天凌晨2点备份
0 2 * * * /path/to/backup.sh >> /var/log/labor-backup.log 2>&1
```

## 六、常见问题

### 6.1 数据库锁定

SQLite并发写入可能锁定，建议：
- 避免高并发写入
- 考虑迁移到PostgreSQL/MySQL

### 6.2 文件权限

```bash
# 确保数据库目录可写
chmod 755 server/data
chown -R www-data:www-data server/data
```

### 6.3 跨域问题

生产环境使用Nginx反向代理，无需额外CORS配置。
