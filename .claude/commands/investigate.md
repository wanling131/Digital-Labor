# Bug 根因分析流程

遇到 Bug 或报错时，执行四阶段深度分析。

## 第一阶段：Reproduce（复现）

1. 记录问题发生的环境
2. 记录触发问题的具体操作步骤
3. 收集错误信息（错误消息、堆栈跟踪、日志）
4. 尝试在干净环境中复现

**输出**：
- [ ] 问题复现步骤已记录
- [ ] 错误信息已收集

---

## 第二阶段：Isolate（隔离）

1. **模块隔离**：确定问题发生在哪个模块
2. **函数隔离**：确定问题发生在哪个函数
3. **行隔离**：确定问题发生在哪行代码

**方法**：二分法注释代码、添加日志观察数据流

---

## 第三阶段：Root Cause（根因）

### 常见根因分类
| 类型 | 示例 |
|------|------|
| 数据问题 | 空值、类型错误、边界条件 |
| 逻辑问题 | 条件判断错误、循环边界错误 |
| 并发问题 | 竞态条件、死锁 |
| 配置问题 | 环境变量、配置项缺失 |

### 5 Whys 分析
连续问5次"为什么"直到找到根因

---

## 第四阶段：Fix（修复）

1. 编写复现该 Bug 的测试用例
2. 编写修复代码
3. 运行测试确认通过
4. 运行完整测试套件确认无回归

---

## 禁止行为

- 盲目试错、随机修改代码
- 只修复表面症状，不找根因
- 修复后不运行测试验证
